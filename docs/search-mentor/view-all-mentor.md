        <h2>Mentors</h2>  
<p>The Mentors page provides users with a list of available mentors, displaying their name, experience, expertise, and other relevant details. Users can browse the list, search for specific mentors, and view their profiles.</p>

<h2>Use Case Scenario</h2>  
<table border="1">
    <tr>
        <th>Use Case</th>
        <th>Mentors Page</th>
    </tr>
    <tr>
        <th>Actor(s)</th>
        <td>Mentee</td>
    </tr>
    <tr>
        <th>Goal</th>
        <td>Allow users to browse, search, and select mentors based on their expertise and experience.</td>
    </tr>
    <tr>
        <th>Preconditions</th>
        <td>User must have a registered mentee account and be on the mentors listing page.</td>
    </tr>
    <tr>
        <th>Main Scenario</th>
        <td>
            1. User navigates to the "Mentors" page.<br>
            2. The system displays a list of mentors with their profile pictures, names, experience, and expertise.<br>
            3. User can scroll through the list to explore different mentors.<br>
            4. User can use the search bar to find a specific mentor based on name or expertise.<br>
            5. User selects a mentor to view their detailed profile.<br>
            6. The system redirects the user to the mentor’s profile page, showing more information and mentorship options.
        </td>
    </tr>
    <tr>
        <th>Outcome</th>
        <td>User successfully finds and selects a mentor for further guidance.</td>
    </tr>
</table>
