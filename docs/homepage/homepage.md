    <h2>Homepage (Public View)</h2>  
    <p>
        The homepage serves as the <strong>main landing screen</strong> for users who are <strong>not yet logged in</strong>. 
        It provides options to <strong>Sign In</strong> or <strong>Register</strong> to access the platform. 
        Once logged in, users are redirected to their respective <strong>role-based homepages</strong>:
    </p>    
    <ul>
        <li><strong>Mentor Homepage</strong> – Access mentoring tools, schedules, and mentee management.</li>
        <li><strong>Mentee Homepage</strong> – View assigned mentors, upcoming sessions, and learning resources.</li>
        <li><strong>Admin Homepage</strong> – Manage users, oversee system activity, and maintain platform settings.</li>
    </ul>  
    <h2>Use Case Scenario</h2> 
    <table border="1">
        <tr>
            <th>Use Case</th>
            <th>Accessing the Public Homepage</th>

        </tr>
        <tr>
            <th>Goal</th>
            <td>Sign in or register to access the platform.</td>
        </tr>
        <tr>
            <th>Preconditions</th>
            <td>The user has <strong>not yet logged in</strong>.</td>
        </tr>
        <tr>
            <th>Main Scenario</th>
            <td>
                1. User opens the application.<br>
                2. User is presented with the <strong>Homepage screen</strong>.<br>
                3. User can choose to <strong>Sign In</strong> (if they already have an account).<br>
                4. If the user is <strong>new</strong>, they can select <strong>Register</strong> to create an account.<br>
                5. Upon successful login/registration, the user is redirected to their <strong>role-specific homepage</strong>.<br>
            </td>
        </tr>

            <th>Outcome</th>
