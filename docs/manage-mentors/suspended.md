        <h2>Suspended Mentors</h2>  
        <p>This displays all the suspended mentors. This enables the admin to keep on track about the mentors' suspension.</p>
        <h2>Use Case Scenario</h2> 
        <table border="1">
    <tr>
        <th>Use Case</th>
        <th>Lifting Suspensions</th>
    </tr>
    <tr>
        <th>Actor(s)</th>
        <td>Admin and Mentors</td>
    </tr>
    <tr>
        <th>Goal</th>
        <td>Navigate on the list of suspended mentors to do actions such as lifting a suspension.</td>
    </tr>
    <tr>
        <th>Preconditions</th>
        <td>The user logged-in to manage suspensions must have an admin access.</td>
    </tr>
    <tr>
        <th>Main Scenario</th>
        <td>
            1. As soon as the admin enters this page, navigation to all suspended accounts would be possible.<br>
            2. Clicking the mentor block/bar containg small details about the mentor will redirect to mentors full detail.<br>
            3. In there, admin can check the mentor and can lift a suspension.<br>
            4. Once the suspension is lifted, system will update the status of the mentor.<br>
            5. Mentor will be notified and regains access to the application once again.<br>
        </td>
    </tr>
    <tr>
        <th>Outcome</th>
        <td>The admin lifts the suspension and the number of active mentors increased <br>while the mentor can now access the application and continue on mentorship.</td>
    </tr>
</table>
