        <h2>Active Mentors</h2>  
        <p>This page displays all the active mentors within the application for management.</p>
        <h2>Use Case Scenario</h2> 
        <table border="1">
    <tr>
        <th>Use Case</th>
        <th>Active Mentors Navigation</th>
    </tr>
    <tr>
        <th>Actor(s)</th>
        <td>Admin</td>
    </tr>
    <tr>
        <th>Goal</th>
        <td>Navigation for the management of different mentors.</td>
    </tr>
    <tr>
        <th>Preconditions</th>
        <td>The user logged in must have an admin access.</td>
    </tr>
    <tr>
        <th>Main Scenario</th>
        <td>
            1. Admin chooses a mentor to view and manage.<br>
            2. Clicking the bar showing some mentor details will redirect the admin to the details of the mentor.<br>
            3. Clicking the triple dots in the right side of the bar will also let the admin choose an action.<br>
            4. After that, admin can now archive or suspend a mentor.<br>
            5. The system will update the mentor status accordingly.
            6. Mentor will be notified.
        </td>
    </tr>
    <tr>
        <th>Outcome</th>
        <td>The admin can control the access of all the active mentors within the application.</td>
    </tr>
</table>
